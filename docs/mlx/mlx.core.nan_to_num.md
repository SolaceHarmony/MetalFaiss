# mlx.core.nan_to_num<a
href="https://ml-explore.github.io/mlx/build/html/#mlx-core-nan-to-num"
class="headerlink" title="Link to this heading">#</a>

## Curated Notes

- Use `nan_to_num` to sanitize tensors before sensitive reductions (e.g., loss averages) or linalg.
- Broadcastable scalars: `nan`, `posinf`, `neginf` accept Python floats; per‑element control can be done via `mx.where` if needed.
- No `device=`; route via default/scoped device or per‑op `stream`.

### Examples

```python
import mlx.core as mx

x = mx.array([0.0, mx.nan, mx.inf, -mx.inf])
y = mx.nan_to_num(x, nan=0.0, posinf=1e3, neginf=-1e3)

# With where for per‑element policy
mask = mx.isfinite(x)
z = mx.where(mask, x, 0.0)
```
